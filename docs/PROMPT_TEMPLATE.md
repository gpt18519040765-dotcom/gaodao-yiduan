# Codex 任务模板

以后给 Codex 发任务时，建议使用以下格式。

## 目标

说明这次希望完成什么。

## 当前问题

说明现在哪里不对、哪里不够好、用户看到的现象是什么。

## 需要修改的页面

列出页面路径或模块，例如：

- `/divination`
- `/result`
- `/learn`
- `components/`
- `lib/divination.ts`

## 交互要求

说明用户应该如何操作，操作后应该看到什么反馈。

## 视觉要求

说明页面风格、动效、布局、颜色、移动端适配要求。

## 不允许做的事

说明本次不要改哪些内容，例如：

- 不改数据库结构。
- 不改起卦算法。
- 不新增大型依赖。
- 不重构无关页面。

## 验收标准

说明怎样算完成，例如：

- 页面能正常打开。
- 移动端不溢出。
- 按钮有反馈。
- `npm run lint` 通过。
- `npm run build` 通过。
