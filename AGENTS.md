# 高导易断

## 项目概况

- 项目名称：高导易断
- 技术栈：Next.js App Router、TypeScript、Tailwind CSS、Framer Motion
- 当前目标：先做好网站 MVP，保持代码简洁，为未来 iOS App 和小程序预留清晰架构。

## 代码要求

- 优先复用组件，不要重复写大段 JSX。
- 页面逻辑和业务逻辑分离。
- 起卦算法放在 `lib/divination.ts`。
- 卦象数据放在 `data/gua.ts`。
- UI 组件放在 `components/`。
- 页面只负责组合组件、传递数据和处理轻量页面状态。
- 每次改动后必须运行 `npm run lint` 和 `npm run build`。
- 不要引入不必要的大型依赖。
- 动画优先使用 Framer Motion 和 CSS，不要过早使用复杂 3D。
- 代码要短、清晰、可维护。
- 新增抽象前先确认是否真的减少重复或降低复杂度。
- 不要一次性重构太多文件，优先小步改进。

## 交互要求

- 页面要有东方仪式感。
- 动画要细腻，不要花哨。
- 移动端优先适配。
- 所有按钮都要有 hover、press、loading 或过渡反馈。
- 起卦流程要有明确步骤感。
- 交互反馈要帮助用户理解当前处于哪一步、下一步做什么。

## 架构方向

- 业务逻辑尽量沉淀到 `lib/` 和 `data/`，避免写死在页面里。
- 组件尽量保持纯粹，方便未来迁移到 iOS App 或微信小程序时复用交互结构。
- 页面文案、课程内容、卦象文本和占断数据应逐步数据化，不要长期散落在 JSX 中。
- API 路由负责服务端读取和聚合数据，前端页面负责展示。

## 输出要求

- 每次完成任务后总结修改了哪些文件。
- 如果发现架构问题，先说明再修改。
- 不要一次性重构太多文件。
- 如果某次任务只要求新增文档，不要改动现有功能代码。
